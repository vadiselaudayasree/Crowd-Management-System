# Crowd Management System

This project is a web-based Crowd Management System developed to handle large gatherings in places like temples, malls, events, and festivals. In many real-life situations, people tend to gather at the same time, which leads to long queues, overcrowding, and sometimes safety issues. This system is designed to reduce that problem by managing the flow of visitors in a better and more organized way.

The main idea of this project is to allow users to book a time slot before visiting a place. Instead of everyone coming at once, visitors are divided into different slots with a limited capacity. This helps in reducing crowd density and improves the overall experience.

The system also keeps track of when a visitor enters and exits using an entry-exit log. This helps the admin know how many people are currently inside the venue. Based on this data, the system can also help in identifying peak hours and managing the crowd more effectively.

There are different modules in the system such as visitor management, slot booking, venue and zone management, staff handling, incident reporting, and notifications. Admins can monitor all activities, handle issues, and ensure smooth operations.

This project was built using Java, Spring Boot, MySQL, HTML, CSS, and JavaScript. It demonstrates how a real-world problem like crowd control can be solved using a structured database and a full-stack approach.

Overall, this system helps in reducing overcrowding, improving safety, and making crowd management more efficient.
